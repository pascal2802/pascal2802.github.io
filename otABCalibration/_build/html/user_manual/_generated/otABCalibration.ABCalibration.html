ABCalibration¶

class ABCalibration(model, observedParameterIndices, toCalibrateParameterIndices, observedOutputIndices, inputObservations, outputObservations, inputPriorDistribution, doeSize, posteriorSampleTargetedSize, criteriaSelection=None, computeABCCriteria=None, n_cpus=1, doeEvalSize=None, referenceValuesYobs=None)¶

ABC calibration algorithm

Available constructors:

Parameters:
modelFunction

The function of the model to be calibrated

observedParameterIndices1-d sequence of integer

Observed parameter indices of the model - provide [] if no observed parameters

toCalibrateParameterIndices1-d sequence of integer

parameter to calibrate indices of the model

observedOutputIndices1-d sequence of integer

observed variables indices of point return by model function

inputObservationsSample

The sample of the observed parameters (2-d sequence of float) provide ot.Sample([[]]) if none parameters is observed

outputObservationsSample

The sample of the observed output variables of the models (2-d sequence of float)

inputPriorDistributionDistribution

Composed prior distribution between parameter to calibrate distribution and obseved parameters uncertainty. Marginals must be in the same order than the model input.

doeSizeint

Sample size of each Monte Carlo experiment

posteriorSampleTargetedSizeint

Target Size of the posterior Conditional Sample

computeABCCriteriaPython function, optional

python function taking 2 argument (Sample predictionSample, Sample outputObservation) This function returns a point (Point) with computed selection criteria Default : default function computing CvRMSE and NMBE for each observed output

criteriaSelectionInterval, optional

Intervals that need to contain computed criteria to consider the model as calibrated Default : CvRMSE < 0.2 and -0.05 < NMBE < 0.05

n_cpusint, default = 1

Number of cpus on which func should be distributed using multiprocessing. If -1, it uses all the cpus available. If 1, it does nothing. Note that you should enforce the multiprocessing guidelines to enable this option. For example on Windows, the entry point of your program should be protected using the if __name__== ‘__main__’ idiom.

doeEvalSizeint, default = None

Number of candidate point to be evaluated among design of experiment. By defaut all points are evaluated, have only an effect if doeEvalSize < doeSize. Can be Usefull if the same seed is set to generated to global doe and if result provided by model function are stored and are only read is evaluation of same point is requested. Enable thus to evaluate sequentially a large DOE until targetPosteriorSample size is met.

referenceValuesYobsPoint, optional

list of values use as 0 values for observed variables. Used to compute relative residuals. Default : None, reference value is taken as 0 for all observed variables.

buildAdditionalModelOutputFunction(additionalOutputIndices)¶

Builds an additional output function for the model based on specified indices.

This method constructs and returns an OpenTURNS function that computes additional outputs from the model’s standard output. The additional outputs are specified by the additionalOutputIndices parameter, which should be indices of outputs already present in the model’s standard output.

Parameters:
additionalOutputIndiceslist of int

A list of indices specifying which components of the model’s standard output should be included in the additional output function. Each index must correspond to a valid output in the model’s standard output.

Returns:
fopenturns.Function

An OpenTURNS function that takes the model’s inputs and returns the specified additional outputs from the model’s standard output.

Notes

The indices provided in additionalOutputIndices must be valid indices of the model’s standard output. Invalid indices will result in an error.

buildResidualFunctionOptimalPoint(indexTrain=None)¶

Builds a residual function for model calibration based on specified training indices.

This method constructs and returns an OpenTURNS function that computes residuals between model predictions and observed outputs for ABC calibration purposes. The function takes observed parameters and parameters to calibrate as input and returns residuals. The training indices parameter allows to specify which observations should be used for building the residual function.

Parameters:
indexTrainlist of int, optional

Indices of observations to be used for building the residual function. If None, all observations will be used. Default is None.

Returns:
fPythonFunction

An OpenTURNS function that computes residuals between model predictions and observed outputs for the specified training observations.

Notes

The indices provided in indexTrain must be valid indices of the observations available in the output observations. Invalid indices will result in an error.

The residual function is built using the FuncResidualFunctionOptimalPoint class and then wrapped in a PythonFunction for parallel evaluation using the specified number of CPUs.

exportCalibrationWithPickle(fileName)¶

This enables future reloading, as the object inherits from an OpenTURNS class that lacks native pickle support - a common issue when Python classes inherit from OpenTURNS classes. Upon reloading, the result can be regenerated using the run method, a process that is nearly instantaneous due to the caching mechanisms of the memoize function, which is exported alongside the object.

Parameters:
filenamestr

pickle object filename export Example : /path/to/filename.pkl
